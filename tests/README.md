# Cabo Game 测试用例说明

## 测试框架

本项目使用 Jest 作为测试框架。

## 安装依赖

```bash
npm install
```

## 运行测试

```bash
# 运行所有测试
npm test

# 监听模式运行测试
npm run test:watch

# 生成测试覆盖率报告
npm run test:coverage
```

## 测试文件结构

```
tests/
├── game-engine.test.js    # 游戏引擎测试
├── card-utils.test.js     # 卡牌工具函数测试
└── score-utils.test.js    # 计分工具函数测试
```

## 测试特点

1. **全面性**: 覆盖所有核心功能模块
2. **场景化**: 基于Spec文档的场景设计测试用例
3. **边界测试**: 包含边界条件和异常情况测试
4. **集成测试**: 包含系统集成测试验证功能协作
5. **清晰命名**: 测试用例命名清晰，易于理解

## 测试数据

测试用例使用了各种测试数据：
- 正常数据：标准游戏场景
- 边界数据：空数组、null、undefined
- 异常数据：无效索引、超出范围值
- 特殊数据：平局、100分重置等特殊场景

## 持续集成

可以将测试集成到CI/CD流程中：
```bash
npm test -- --coverage --ci
```

## 测试报告

运行测试后，Jest会生成详细的测试报告，包括：
- 测试通过/失败数量
- 测试覆盖率
- 失败测试的详细信息
