# 修复开局偷看牌Bug Spec

## Why
当前开局偷看牌功能存在两个bug：
1. 确认查看后，个人牌堆的牌应该保持暗置，而不是明牌展示
2. 玩家选择牌后不允许取消选择重新选择，一旦翻开就不能翻回

## What Changes
- 修复偷看阶段确认后牌的显示状态：确认查看后牌应暗置，仅在倒计时期间显示
- 修复选择逻辑：选择牌后不允许取消，翻开后不可翻回

## Impact
- Affected specs: 开局偷看牌阶段
- Affected code: 
  - `utils/game-engine.js` - 偷看牌逻辑
  - `pages/game/game.js` - 偷看牌交互
  - `pages/game/game.wxml` - 偷看牌UI

## ADDED Requirements

### Requirement: 偷看牌选择限制
系统应限制玩家在选择牌时的操作，一旦选择不可取消。

#### Scenario: 选择牌后不可取消
- **WHEN** 玩家在偷看阶段点击选择一张牌
- **THEN** 该牌被选中并翻转显示，不允许再次点击取消选择
- **AND** 玩家最多只能选择2张牌

#### Scenario: 确认查看后牌暗置
- **WHEN** 玩家确认查看选中的2张牌
- **THEN** 进入倒计时阶段，显示选中的牌供玩家记忆
- **AND** 倒计时结束后，所有牌暗置回原位置，进入正常游戏
- **AND** 个人牌堆区域显示为暗牌状态（玩家已知但不明牌展示）

## MODIFIED Requirements

### Requirement: 开局偷看牌
系统应支持开局偷看牌功能，玩家可选择2张暗牌查看3秒后自动暗置。

#### Scenario: 开局偷看牌流程
- **WHEN** 游戏开始后，进入偷看阶段
- **THEN** 系统提示玩家"你可以选择查看2张牌"，玩家点击选择2张暗牌
- **AND** 选中的牌翻转显示，不可取消选择
- **AND** 玩家确认后进入查看阶段，倒计时3秒
- **AND** 3秒后，牌自动暗置回原位置，进入正常游戏回合
- **AND** 游戏界面中玩家的手牌显示为暗牌（背面），但系统记录玩家已知道这2张牌

## REMOVED Requirements
无移除的需求。
